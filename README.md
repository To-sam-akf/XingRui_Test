# XingRui Test

本项目包含两道 Python 编程题：第一题完成债券公开数据的采集与 CSV
导出，第二题实现一个由正则配置驱动的文本信息提取函数。第一题使用
`requests` 简化 HTTP 请求，其余数据处理主要使用 Python 标准库完成。

## 运行环境

- Python 3.11 及以上版本
- 可访问中国货币网的网络环境

## 第一题：债券数据采集

### 题目需求

从[中国货币网债券信息页面](https://www.chinamoney.com.cn/english/bdInfo/)
获取公开债券数据，并按以下条件筛选：

- Bond Type：`Treasury Bond`
- Issue Year：`2023`

程序需要提取以下字段，并将结果保存为有效的 CSV 文件：

| CSV 列名 | 字段含义 |
| --- | --- |
| ISIN | 国际证券识别码 |
| Bond Code | 债券代码 |
| Issuer | 发行人 |
| Bond Type | 债券类型 |
| Issue Date | 发行日期 |
| Latest Rating | 最新评级 |

### 题目背景

债券市场公开信息通常分布在金融信息平台中。将网页查询过程程序化，可以减少
人工筛选和复制数据的工作，并为后续的数据清洗、统计分析和定期更新提供统一的
结构化数据源。

本题的核心不只是发起一次网络请求，还需要识别页面使用的数据接口、转换查询
条件、处理分页结果，并保证最终文件可以被 Excel、数据库或数据分析工具正常
读取。

### 实现说明

程序文件为 `First.py`，主要流程如下：

1. 请求查询条件接口，根据页面显示名称 `Treasury Bond` 获取网站内部使用的
   债券类型代码。
2. 携带债券类型和发行年份等参数请求债券列表接口。
3. 根据接口返回的总页数逐页获取数据，避免遗漏记录。
4. 校验实际获取数量是否与接口声明的总记录数一致。
5. 从返回结果中选择题目要求的六个字段，缺失值统一写为 `---`。
6. 使用 UTF-8 BOM 编码输出 CSV，提升文件在 Excel 中打开时的兼容性。

网络请求设置了超时时间和最多三次重试，并在重试间隔及分页请求之间加入短暂
等待，以增强程序面对临时网络异常时的稳定性。

### 使用的技术栈

- Python 3.11
- `requests`：复用 HTTP 会话、发送 POST 请求并解析 JSON 响应
- `csv`：生成标准 CSV 文件
- `argparse`：处理命令行参数
- `pathlib`：管理输出文件路径
- 类型注解：说明函数输入、输出及数据结构

### 运行方式

使用默认文件名运行：

```bash
python3 First.py
```

指定输出路径：

```bash
python3 First.py --output data/treasury_bonds_2023.csv
```

默认结果保存在 `treasury_bonds_2023.csv`。当前示例文件包含表头及 111 条债券
记录。

## 第二题：自定义正则匹配

### 题目需求

实现 `reg_search(text, regex_list)` 函数：

- `text`：需要提取信息的文本。
- `regex_list`：正则表达式配置列表，每个字典中的键为结果字段名，值为该字段
  对应的正则表达式。
- 返回值：按配置顺序生成的匹配结果列表。

示例文本为债券募集说明书片段，需要提取标的证券代码和换股期限，预期结果为：

```python
[
    {
        "标的证券": "600900.SH",
        "换股期限": ["2023-06-02", "2027-06-01"],
    }
]
```

### 题目背景

债券公告、募集说明书等金融文档多为非结构化或半结构化文本。同一份文档中可能
包含证券代码、期限、日期等重要信息，人工查找效率较低。通过可配置的正则表达式
提取字段，可以在不修改核心函数的情况下适配不同文本和业务字段，是一种轻量的
文档结构化方案。

### 实现说明

程序文件为 `second.py`。`reg_search` 会依次处理每组配置，并对每个字段执行
以下操作：

1. 使用 `re.DOTALL` 编译正则，使表达式能够匹配跨行文本。
2. 如果正则包含一个捕获组，返回捕获组内容；没有捕获组时返回完整匹配内容。
3. 使用 `finditer` 获取字段的全部匹配结果。
4. 将中文日期（如 `2023 年 6 月 2 日`）标准化为 `2023-06-02`。
5. 单个匹配值返回字符串，多个匹配值返回列表，没有匹配时返回空列表。

为避免结果含义不明确，每个正则最多允许一个捕获组；仅用于组合匹配规则的括号
应写成非捕获组 `(?:...)`。

### 使用的技术栈

- Python 3.11
- `re`：正则表达式编译、全文搜索和日期识别
- `typing.TypeAlias`：定义正则配置和返回结果的数据类型
- 类型注解：约束函数接口并提高代码可读性

### 运行方式

运行内置示例：

```bash
python3 second.py
```

也可以在其他模块中导入函数：

```python
from second import reg_search

result = reg_search(text, regex_list)
```
